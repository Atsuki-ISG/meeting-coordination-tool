## 2. 認証機能

### 2.1 ログイン

**URL**: `/login`

**機能概要**:
- Googleアカウントでログイン
- 初回ログイン時にメンバーとして自動登録（**承認待ち状態**）
- Googleカレンダーへのアクセス権限を取得

**処理フロー**:
1. 「Googleでログイン」ボタンをクリック
2. Google OAuth同意画面にリダイレクト
3. 権限を承認
4. コールバック処理でメンバー登録/更新
5. **新規ユーザー** → 承認待ち画面（`/pending-approval`）
6. **承認済み・チーム所属あり** → ダッシュボード
7. **承認済み・チーム所属なし** → チーム画面

**取得スコープ**:
- `calendar.readonly` - カレンダー参照
- `calendar.events` - イベント作成・削除

### 2.2 承認待ち画面

**URL**: `/pending-approval`

**機能概要**:
- 新規ユーザーがシステム管理者の承認を待つ画面
- 30秒ごとに自動でステータスを確認
- 手動で「ステータスを確認」ボタンも使用可能
- 承認されると自動的にチーム画面またはダッシュボードにリダイレクト

### 2.3 ログアウト

**URL**: `/api/auth/logout`

**機能概要**:
- セッションCookieを削除
- ログインページにリダイレクト

## 3. システム管理者機能

### 3.1 ユーザー承認

**URL**: `/admin`

**機能概要**:
- 承認待ちユーザーの一覧表示
- ユーザーの承認/拒否
- 全ユーザーの管理（有効化/停止）
- システム管理者権限の付与/解除

**ユーザーステータス**:
| ステータス | 説明 |
|-----------|------|
| pending | 承認待ち。ログインはできるがアプリは使用不可 |
| active | 有効。通常利用可能 |
| suspended | 停止。ログイン不可 |

**システム管理者の設定方法**:
1. 環境変数 `SYSTEM_ADMIN_EMAILS` にメールアドレスを設定
2. 該当ユーザーがログインすると自動的にシステム管理者になる
3. または、既存のシステム管理者が `/admin` で権限を付与

### 3.2 メンテナンスモード

**機能概要**:
- 有効化すると外部ユーザーからの予約を停止
- 社内メンバーは引き続きアクセス可能

### 3.3 API利用量監視

**表示項目**:
- 総リクエスト数
- 空き枠確認回数
- 予約作成回数
- キャンセル回数

## 4. チーム機能

### 4.1 チーム作成

**URL**: `/team`

**機能概要**:
- 新しいチームを作成
- 8文字の招待コードを自動生成

**入力項目**:
| 項目 | 必須 | バリデーション |
|------|------|---------------|
| チーム名 | 必須 | 1文字以上 |

**出力**:
- 招待コード（例: `AB3D5F7H`）

### 4.2 チーム参加

**URL**: `/team`

**機能概要**:
- 招待コードでチームに参加

**入力項目**:
| 項目 | 必須 | バリデーション |
|------|------|---------------|
| 招待コード | 必須 | 8文字 |

**エラー**:
- 無効な招待コード
- 既に別チームに所属

### 4.3 チーム管理

**URL**: `/settings/team`

**対象**: チーム管理者（role: admin）

**機能概要**:
- チーム名の編集
- 招待コードの確認・コピー
- メンバーの権限変更（管理者/メンバー）
- メンバーの削除
- チームの削除

## 5. 予約メニュー管理

### 5.1 一覧表示

**URL**: `/event-types`

**表示項目**:
- タイトル
- 所要時間
- 予約ページURL
- 有効/無効ステータス

**操作**:
- 新規作成
- 編集
- リンクコピー
- 予約ページを開く

### 5.2 新規作成

**URL**: `/event-types/new`

**入力項目**:
| 項目 | 必須 | バリデーション | デフォルト |
|------|------|---------------|-----------|
| タイトル | 必須 | 1文字以上 | - |
| 説明 | 任意 | - | - |
| 所要時間 | 必須 | 15/30/45/60分 | 30分 |
| 参加メンバー | 任意 | チーム内メンバー | 作成者のみ |

**処理**:
- スラッグを自動生成（UUID）
- 作成者を主催者として設定

### 5.3 編集

**URL**: `/event-types/{id}`

**編集可能項目**:
- タイトル
- 説明
- 所要時間
- 参加メンバー

**制約**:
- スラッグは変更不可
- 同一チームのメンバーのみ編集可能

### 5.4 削除

**処理**:
- 予約メニューを非アクティブ化
- 関連する予約は維持（履歴として）

## 6. 予約ページ

### 6.1 カレンダー形式の空き枠表示

**URL**: `/book/{slug}`

**機能概要**:
- **月間カレンダー形式**で空き枠を表示
- 各日付に「最早時間」と「残り枠数」を表示
- 日付をクリックすると詳細な時間枠を表示

**カレンダー表示**:
```
┌───────┬───────┬───────┬───────┬───────┬───────┬───────┐
│  日   │  月   │  火   │  水   │  木   │  金   │  土   │
├───────┼───────┼───────┼───────┼───────┼───────┼───────┤
│   5   │   6   │   7   │   8   │   9   │  10   │  11   │
│ 09:00 │ 10:00 │ 09:30 │   ー  │ 09:00 │ 11:00 │   ー  │
│ 残り5 │ 残り3 │ 残り2 │       │ 残り7 │ 残り1 │       │
└───────┴───────┴───────┴───────┴───────┴───────┴───────┘
```

**表示要素**:
- 予約メニュー情報（タイトル、説明、所要時間）
- 月間カレンダー（前月・次月への移動可能）
- 日付ごとの最早時間と残り枠数
- 選択した日付の時間枠一覧

**空き枠計算ロジック**:
1. 参加メンバー全員のGoogleカレンダーを取得
2. 予約済み時間を除外
3. 予約可能時間設定（曜日・時間）でフィルタ
4. 所要時間単位でスロット化

### 6.2 予約フォーム

**入力項目**:
| 項目 | 必須 | バリデーション |
|------|------|---------------|
| お名前 | 必須 | 1文字以上 |
| メールアドレス | 必須 | メール形式 |
| ご相談内容・備考 | 必須 | 1文字以上 |

### 6.3 予約確定

**処理フロー**:
1. 空き状況を再確認（競合防止）
2. Googleカレンダーにイベント作成
3. Google Meetリンクを自動生成
4. 予約データをDBに保存
5. キャンセルトークンを生成

**確定画面の表示**:
- 予約日時
- Google Meetリンク
- キャンセルURL

**Googleカレンダーイベント**:
- タイトル: `{予約メニュー名} - {予約者名}`
- 参加者: 全メンバー + 予約者
- Google Meet: 自動付与
- 説明: 備考を含む

## 7. キャンセル機能

### 7.1 キャンセルページ

**URL**: `/cancel/{token}?bookingId={id}`

**表示内容**:
- 予約詳細（日時、予約メニュー名）
- キャンセル確認ボタン

**制約**:
- 過去の予約はキャンセル不可
- 既にキャンセル済みの場合はエラー

### 7.2 キャンセル処理

**処理フロー**:
1. トークンの検証
2. Googleカレンダーのイベント削除
3. 予約ステータスを「キャンセル」に更新
4. キャンセルトークンを無効化

## 8. 予約可能時間設定

### 8.1 曜日別設定

**URL**: `/settings`

**設定項目（曜日ごと）**:
| 項目 | 説明 |
|------|------|
| 有効/無効 | その曜日に予約を受け付けるか |
| 開始時間 | 予約受付開始時間（30分刻み） |
| 終了時間 | 予約受付終了時間（30分刻み） |

**デフォルト値**:
- 月〜金: 有効（09:00〜18:00）
- 土・日: 無効

## 11. レート制限

### 11.1 短期レート制限

**対象API**: `/api/availability`, `/api/bookings`, `/api/bookings/cancel`

**制限**:
- 同一IPから10秒間に5リクエストまで
- 超過時は `429 Too Many Requests` を返却

**目的**:
- DoS的なアクセスの防止
- Google Calendar API の過剰消費防止

### 11.2 月間使用量制限

**制限**:
- 環境変数 `USAGE_ALERT_THRESHOLD` で設定（デフォルト: 800,000）
- 上限到達時は `429 Too Many Requests` を返却

**監視**:
- 80%到達時に警告（Google Chat通知）
- `/admin` 画面で使用量を確認可能

## 12. API仕様

### 12.1 空き枠取得

**エンドポイント**: `GET /api/availability`

**パラメータ**:
| 名前 | 型 | 必須 | 説明 |
|------|-----|------|------|
| eventTypeId | UUID | 必須 | 予約メニューID |
| daysAhead | number | 任意 | 取得日数（デフォルト: 14） |

**レスポンス**:
```json
{
  "slots": [
    { "start": "2024-01-15T09:00:00+09:00", "end": "2024-01-15T09:30:00+09:00" },
    { "start": "2024-01-15T10:00:00+09:00", "end": "2024-01-15T10:30:00+09:00" }
  ],
  "timezone": "Asia/Tokyo",
  "eventType": {
    "title": "初回相談",
    "description": "...",
    "durationMinutes": 30
  }
}
```

### 12.2 予約作成

**エンドポイント**: `POST /api/bookings`

**リクエスト**:
```json
{
  "eventTypeId": "uuid",
  "startAt": "2024-01-15T09:00:00+09:00",
  "endAt": "2024-01-15T09:30:00+09:00",
  "name": "山田太郎",
  "email": "yamada@example.com",
  "note": "ご相談内容"
}
```

**レスポンス**:
```json
{
  "success": true,
  "booking": { "id": "uuid", "startAt": "...", "endAt": "..." },
  "cancelUrl": "https://example.com/cancel/xxx?bookingId=yyy",
  "meetLink": "https://meet.google.com/xxx"
}
```

### 12.3 キャンセル

**エンドポイント**: `POST /api/bookings/cancel`

**リクエスト**:
```json
{
  "bookingId": "uuid",
  "token": "cancel-token"
}
```

**レスポンス**:
```json
{
  "success": true,
  "message": "予約がキャンセルされました。"
}
```

### 12.4 ユーザーステータス確認

**エンドポイント**: `GET /api/auth/status`

**レスポンス**:
```json
{
  "status": "active",
  "isSystemAdmin": true,
  "teamId": "uuid"
}
```
